## **41\. Wireframe design principles for MVP**

### **Principle 1: clarity over beauty**

The first version should prioritise usability and trust, not visual complexity.

### **Principle 2: role separation**

Tutor, parent, student, and admin interfaces should feel clearly different.

### **Principle 3: short paths**

Core actions should take as few clicks as possible.

### **Principle 4: dashboard discipline**

Do not overload dashboards. Each should surface only the most useful information.

### **Principle 5: make invitations easy**

Invite flows are critical for early growth and must be frictionless.
